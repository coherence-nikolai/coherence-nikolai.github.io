# Hidden Worlds Phase Map

## Protected Core

Every Person Lives in a World You Cannot See.

Any feature that makes the app feel certain, diagnostic, therapeutic, gamified, or narrowly school/work focused should be removed or rewritten.

## Assumptions

- Hidden Worlds is separate from the Tone family, but should feel like it belongs in the same world: dark, quiet, instrument-like, restrained, and emotionally precise.
- The product should be useful for work, family, love, friendship, money, body, culture, grief, digital life, public space, strangers, and change.
- The app should not claim to know why someone behaved a certain way. It should offer possible worlds and better questions.
- The web app is the current live product surface. iOS can follow the same IA and visual system.

## Completed Pass

1. Tone reference extraction: use Tone Recall's ritual pacing, dark field, soft glow, and chamber depth without copying its waveform/dial language.
2. IA simplification: make the main loop Lens -> Atlas -> Daily.
3. Feature elimination: remove top-level Swap; fold "same moment, many worlds" into Lens results through alternate possible worlds.
4. Atlas positioning: present the 112-entry library as the comprehensive searchable World Atlas.
5. Safety copy: reduce certainty language, avoid clinical/student/productivity drift, and add stronger guidance for health or unsafe situations.
6. Aesthetic pass: make the path/door mark the Hidden Worlds signature and reduce dashboard-like panels.

## Next Phases

1. Atlas normalization: add rich metadata to every older atlas entry, not only the newest forty.
2. Deep entry view: give each atlas world a fuller reflection page with surface, possible world, what not to assume, boundary, better question, and adjacent prompts.
3. Daily refinement: make Daily feel sacred and minimal, with no streaks or tracking.
4. iOS parity: port the Lens -> Atlas -> Daily structure and Tone-family aesthetic into SwiftUI.
5. AI layer: generate possibilities with strict humility, never diagnosis or certainty.
6. QA and review: run red-team checks from investor, App Store, therapist, educator, parent, and teen perspectives before each major release.

## Current Critique

- The app is now clearer, but the Atlas still needs richer metadata across all 112 worlds to feel truly comprehensive.
- The web app has a stronger instrument feel, but the iOS version still needs a dedicated pass to match this direction.
- The current static Lens is safe and fast, but a future AI layer will need rigorous prompt and evaluation guardrails before it should ship.
